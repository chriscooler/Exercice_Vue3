Flow A — Authentication with PSC
Initial login via Pro Santé Connect → JWT issued to the frontend
Flow B — API Query & Document Access
Every request after login — KrakenD routes based on JWT and request type
👤 User
① Query
🐙 KrakenD
KrakenD checks the JWT and routes to one of three paths ↓
Path A JWT valid + data query
→ RBAC Server ▾
⚙️ RBAC Server → 💾 DB / Cache → ✅ Response

JWT is valid. The request is forwarded to the RBAC Server which fetches user rights from the database or cache, then returns the data.

Path B JWT valid + document request
→ documentd ▾
📄 documentd → 🗃️ S3 Storage → ✅ File / Response

JWT is valid but request targets a document. documentd retrieves the file from S3 storage and streams it back to the user.

Path C JWT missing or expired (cleed)
→ Keycloak re-auth ▾
🔐 Keycloak OIDC → deed (new JWT) → 🐙 KrakenD retry

cleed = re-auth trigger. Keycloak issues a fresh JWT (deed). KrakenD then retries routing the original request.
